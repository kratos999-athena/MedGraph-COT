# MedGraph-CoT: Advanced Medical Hallucination Mitigation

A sophisticated hybrid architecture for medical-grade LLM reliability. This repository implements a multi-layered approach to grounding: combining **Unsloth-optimized fine-tuning**, **Hierarchical GraphRAG** (Neo4j + NetworkX + Leiden), and **Chain-of-Thought (CoT)** reasoning with structured validation.

## Key Features

* **Optimized Fine-Tuning**: Leveraging **Unsloth** for memory-efficient LoRA/QLoRA training on medical datasets.
* **Hierarchical GraphRAG**: 
    * Entity & Relationship extraction via LLM.
    * Storage in **Neo4j** for persistent relational mapping.
    * Memory-optimized community detection using **Leiden** on **NetworkX**.
* **Recursive Summarization**: Top-5 community summarization with **Residual Node Injection** from leftover clusters to prevent data loss.
* **CoT Reasoning Engine**: Multi-stage reasoning path using **Pydantic** for schema-strict, hallucination-free outputs.

## System Architecture

### 1. Fine-Tuning (Unsloth Template)
The model is fine-tuned to understand medical nomenclature and structural formatting using the following LoRA configuration:

| Parameter | Value |
| :--- | :--- |
| **Rank (r)** | 16 |
| **Alpha** | 32 |
| **Target Modules** | q_proj, k_proj, v_proj, o_proj, gate_proj, up_proj, down_proj |
| **Dropout** | 0 |
| **Optimization** | 4-bit quantization (bitsandbytes) |

### 2. Graph Pipeline & Leiden Extraction
To overcome Neo4j free-tier limitations for advanced graph algorithms:
1.  **Extract**: LLM parses raw text into Graph Triplets.
2.  **Load**: Triplets are stored in **Neo4j** for visualization and querying.
3.  **Process**: The graph is exported to **NetworkX** to apply the **Leiden Algorithm**.
4.  **Summarize**: 
    * Identifies communities within the knowledge graph.
    * Generates summaries for the **Top 5** most relevant communities.
    * **Injection Logic**: Remaining nodes from smaller/leftover communities are injected into the context to ensure comprehensive coverage.

### 3. Reasoning & Merge Logic
Final answer generation is gated by a **Chain-of-Thought** process:
* **Pydantic Validation**: Ensures the output follows a strict `{reasoning_path: str, evidence: list, final_answer: str}` schema.
* **Conflict Resolution**: Custom logic to merge findings from the fine-tuned internal weights and the retrieved GraphRAG context.
